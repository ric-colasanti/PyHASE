PyHASE

Axioms

1) For people to understand ABM they need to experience/use an ABM

2) PHASE is trying to get people to use sociological agents. Ie people agents in either networks or geographical environments.

3) The participants do not need to know how to use a generic ABM or fully program a sociological AMB themselves

4) NetLogo has a sharp learning curve just to get a working ABM especially if little or no programming experience

PyPHASE is not a comprehensive ABM it is a wrapper around a Python AMB library. In this case it is a simple toy ABM written for this demo. PyHASE should work with a more comprehensive Sociological ABM python library. It is intended for "workshop" use only. It is not intended as a fully supported ABM IDE

The starting point of PyHASE is the game library PyGame Zero (pgzero). This handels all of the graphics and iterations. pgzero isolates the update function (see code examples). This makes the code much more readable.

pgzero is very lightweight and runs very well on a Raspberry Pi Zero. Used with the Mu editor https://codewith.mu/ could be a very engaging experience for PHASE participants. I have tested this combination on a Raspberry Pi Zero and it works well.

The code is at the github repository https://github.com/ric-colasanti/PyHASE

The PyHASE versions of three examples can be seen in the code link in the navbar above

The initial plan would be that participants would alter boilerplate code, not write full code them selves. For example in the Life demo they could would change the initial patten and see the effect. But they would be waled through the code of the demos. (not the ABM code) This would expose them to the idea of coding an ABM

Note you will need to have pgzero installed

      
        pip install pgzero
